  📋 任务实现计划

  一、需求分析

  根据示例数据，Excel包含150行媒体摘录，每行包含：
  - 序号：行号
  - 媒体名称：如"CCTV-13《走过2025》"
  - 发布时间：2025-12-31等
  - 标题：新闻标题
  - 链接：HTML页面URL（视频或图文）
  - 备注：摘要信息

  目标：将每行转化为一页PPT，包含文字信息和视觉内容（视频截图或图文截图+摘要）

  二、技术方案选型

  推荐使用Python，原因：
  1. 成熟的库支持：pandas、python-pptx、selenium、opencv
  2. 稳健性更好：异常处理、重试机制容易实现
  3. 调试方便：可以逐步测试每个模块

  核心依赖库：
  - pandas + openpyxl - Excel读写
  - python-pptx - PPT生成
  - selenium + ChromeDriver - 网页截图和内容抓取
  - opencv-python - 视频帧提取
  - requests + beautifulsoup4 - HTML解析
  - Pillow - 图片处理

  三、架构设计

  main.py (主流程)
  ├── excel_reader.py (读取Excel)
  ├── content_analyzer.py (分析链接类型：视频/图文)
  ├── video_extractor.py (提取视频关键帧)
  ├── webpage_scraper.py (抓取图文内容+截图)
  ├── ppt_generator.py (生成PPT)
  └── utils.py (工具函数：重试、日志等)

  四、实现步骤

  Task 1: 分析需求和技术方案选型 ✅

  - 已完成数据结构分析
  - 已确定Python技术栈

  Task 2: 设计整体架构

  - 模块化设计，每个功能独立
  - 统一的错误处理和日志
  - 配置文件管理参数（超时、重试次数等）

  Task 3: 实现Excel读取模块

  # 读取指定列：媒体名称、发布时间、标题、链接
  # 数据清洗和验证

  Task 4: 实现HTML内容类型识别

  # 判断链接是否为视频（tv.cctv.com/video特征）
  # 判断是否为图文博客

  Task 5: 实现视频关键帧提取

  策略：
  1. 方案A（推荐）：使用selenium访问视频页面，等待视频加载，截取视频画面
  2. 方案B：如果能获取视频真实URL，使用opencv提取关键帧
  3. 降级方案：如果视频提取失败，使用整个页面截图

  Task 6: 实现图文内容抓取

  策略：
  1. 使用selenium访问页面，等待加载完成
  2. 截取页面主体内容区域
  3. 使用BeautifulSoup提取正文文本
  4. 简单摘要：提取前200-300字作为摘要

  Task 7: 实现PPT生成和排版

  布局设计：
  +--------------------------------+
  | 标题：【媒体名称】标题          |
  | 发布时间：2025-12-31           |
  +--------------------------------+
  | [图片区域]                      |
  |                                |
  | 摘要文字（如果是图文）          |
  +--------------------------------+

  Task 8: 集成测试和优化

  - 先测试前5条数据
  - 添加进度条显示
  - 优化错误处理（单条失败不影响整体）
  - 添加日志记录

  五、稳健性保障措施

  1. 网络请求：
    - 设置合理超时（30秒）
    - 失败重试3次
    - User-Agent伪装避免反爬
  2. 异常处理：
    - 每条数据独立处理，单条失败记录日志继续下一条
    - 所有外部调用都有try-except
  3. 资源管理：
    - selenium浏览器使用完及时关闭
    - 临时图片文件及时清理
  4. 可配置性：
    - 提取的图片质量、尺寸可配置
    - 摘要长度可配置
    - 超时和重试参数可配置

  六、预估时间

  - 基础框架搭建：30分钟
  - 各模块实现：2-3小时
  - 测试和优化：1-2小时
  - 总计：3-5小时
